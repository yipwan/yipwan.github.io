    宏

    TEST_F：用于定义普通的测试用例，适用于需要使用测试套件类中的成员变量和方法的场景。
    TEST_P：用于定义参数化测试用例，适用于需要对一组输入数据运行相同测试逻辑的场景。
    TEST：用于定义独立的测试用例，不依赖于测试套件类。
    TEST_CASE：较新的宏，用于定义测试用例，提供了一些额外的功能。
    INSTANTIATE_TEST_SUITE_P：用于实例化参数化测试套件，定义一组输入数据并运行相同的测试逻辑。
    ASSERT_ 宏
    行为：如果 ASSERT_     宏的断言失败，测试将立即终止，并报告失败。测试框架不会继续执行该测试用例中的后续代码。
    用途：通常用于检查前置条件或关键的假设。如果这些条件不满足，继续执行测试是没有意义的。
    EXPECT_ 宏
    行为：如果 EXPECT_     宏的断言失败，测试不会立即终止。测试框架会记录失败，但会继续执行该测试用例中的后续代码。
    用途：通常用于检查多个条件，即使某些条件失败，也希望能够继续执行后续的检查，以便提供更多的调试信息。

    配置

    --gtest_print_time：控制是否打印每个测试的运行时间。设置为 0 可禁用此功能。
    --gtest_break_on_failure：在调试器下运行测试时，如果断言失败，测试程序会中断并进入交互模式。这对于调试失败的测试非常有用。
    --gtest_filter：通过模式匹配选择要运行的测试。例如，--gtest_filter=FooTest.* 会选择所有属于 FooTest 测试套件的测试。
    --gtest_list_tests：列出所有测试用例，而不运行它们。
    --gtest_internal_run_death_test：用于内部死亡测试的运行。
    --gtest_shuffle：控制是否随机打乱测试用例的运行顺序。
    --gtest_random_seed：设置随机数种子，用于控制测试用例的随机顺序。

    visual studio 中配置Goolge Test

    配置时选择要测试的项目。如果你不选择任何项目，则需要手动添加对要测试的项目的引用

    参考：https://learn.microsoft.com/zh-cn/visualstudio/test/how-to-use-google-test-for-cpp?view=vs-2022 1、 在被测试项目的同级下新建GoogleTest项目 1）将Google Test 用作1、静态库 2、 动态库 被测试项目是静态库则选静态链接，是动态库择选动态链接。如果是exe，需要先在被测项目的属性》配置属性》常规》配置类型 从exe改为静态库 2）C++ 运行时库 1、静态链接 2 动态链接（推荐静态链接）

    2、添加要进行单元测试的头文件，通常，被测试项目在test.cpp的上一级目录。例如被测试项目是 Scip项目

    include "../Scip/branching.h"

    3、编写一个测试用例 TEST(case名, 功能名) { //EXPECT_EQ(1, 0); //EXPECT_TRUE(true); EXPECT_TRUE(被测函数() == true); } 参考写法：https://zhuanlan.zhihu.com/p/369466622

    4、使用测试资源管理器查看测试资源。管理多个case 参考：https://learn.microsoft.com/zh-cn/visualstudio/test/run-unit-tests-with-test-explorer?view=vs-2022

    5、总结: 使用Goolge Test 的作用 1、将测试测试用例从main函数中解放出来 2、通过测试资源管理器直观看测试用例的通过情况、耗时

    参数化测试用例

    INSTANTIATE_TEST_SUITE_P(AdditionTests, AddTest,
                         ::testing::Values(std::make_tuple(2, 3, 5),
                                           std::make_tuple(-1, 1, 0),
                                           std::make_tuple(0, 0, 0)));
这里的testing::Values中的参数可以是任意一种容器，比如list

    mock技术

    FileReader 中提供了一个读取文件字符串的接口

    // FileReader.h
class FileReader {
public:
    virtual std::string ReadFile(const std::string& filename) = 0;
    virtual ~FileReader() {}
};

FileService 中提供了一个文件处理的功能，该功能调用一个FileReader的接口。
// FileService.h
class FileService {
public:
    FileService(FileReader* reader) : reader_(reader) {}

    std::string GetFileContent(const std::string& filename) {
        try {
            return reader_->ReadFile(filename);
        } catch (const std::exception& e) {
            return "Error: " + std::string(e.what());
        }
    }

private:
    FileReader* reader_;
};

    Mock使用。

    // Mock FileReader
class MockFileReader : public FileReader {
public:
    MOCK_METHOD(std::string, ReadFile, (const std::string& filename));
};


TEST(FileServiceTest, GetFileContent) {
    MockFileReader mockReader;
    FileService fileService(&mockReader);

    // 设置期望行为
    EXPECT_CALL(mockReader, ReadFile("test.txt"))
        .WillOnce(testing::Throw(std::runtime_error("File not found")));

    // 调用FileService的方法
    std::string content = fileService.GetFileContent("test.txt");

    // 验证结果
    EXPECT_EQ(content, "Error: File not found");
}
